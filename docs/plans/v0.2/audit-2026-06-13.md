## 3. Accepted / known limitations (documented, not bugs)

## 4. Verified working (regression baseline)

- **Canvas generation** (M0.2.1): KL completion auto-enqueues `run_build_canvas`; produces a well-formed 7-section Canvas with correct per-type `data` keys and narration; `source_signature` set; stale computed on read.
- **Canvas rendering** (M0.2.2): all 9 section components render on-brand; 4-state Overview; Lessons removed from nav + `/lessons` redirects; tree-shaken from bundle.
- **Guided Tour** (M0.2.3): deterministic per-section sequencing — `canvas_section idx=0 → narrate → turn_complete → idx=1` confirmed live; highlight + scroll driven by backend events.
- **Detour/resume** (M0.2.4): 10/10 `TourController` state-machine assertions, incl. the "no resume before answer audio" guard and re-anchor to the correct section.
- **Click-to-jump** (M0.2.4 + R1): live test confirmed the model stops the old section and narrates the jumped-to section's content.
- **Persistence**: Canvas not rebuilt on refresh (`built_at` stable).
